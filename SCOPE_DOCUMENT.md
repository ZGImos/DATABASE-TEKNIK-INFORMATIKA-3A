Dokumen ini menjelaskan **ruang lingkup (scope)** pengembangan website e-commerce fashion. Dokumen ini menjadi acuan utama Project Manager dan seluruh tim selama proyek berlangsung.

---

## 1. Tujuan Proyek

Membangun website e-commerce fashion berbasis web yang memungkinkan pengguna untuk melihat produk fashion, memilih varian (size dan warna), memasukkan produk ke keranjang, melakukan checkout, serta membuat pesanan secara online.

---

## 2. In Scope (Termasuk dalam Proyek)

Fitur-fitur berikut **WAJIB dikerjakan** dalam proyek ini:

### 2.1 Manajemen User

* Registrasi user
* Login user
* Profil user/customer

### 2.2 Manajemen Produk Fashion

* Katalog produk
* Detail produk
* Brand dan kategori produk
* Media/gambar produk
* Varian produk (size dan warna)
* Stok per varian

### 2.3 Keranjang & Pesanan

* Keranjang belanja
* Update jumlah item
* Checkout
* Pembuatan pesanan
* Status pesanan

### 2.4 Pembayaran & Pengiriman

* Pilih metode pembayaran
* Pilih alamat pengiriman
* Pilih jasa pengiriman
* Informasi biaya pengiriman

### 2.5 Fitur Pendukung

* Wishlist produk
* Review dan rating
* Promo dan diskon
* Sistem return barang
* Sistem subscription user

---

## 3. Out of Scope (Tidak Termasuk dalam Proyek)

Fitur berikut **TIDAK dikerjakan** dalam proyek ini:

* Aplikasi mobile (Android/iOS)
* Sistem multi-vendor
* Live chat customer service
* Refund otomatis terintegrasi payment gateway
* Integrasi API real-time pihak ketiga
* Sistem rekomendasi berbasis AI

---

## 4. Target Pengguna

* Customer / pembeli produk fashion
* Admin pengelola toko

---

## 5. Platform & Batasan

* Platform: Website berbasis web
* Sistem bersifat akademik dan simulasi
* Pembayaran dan pengiriman dapat menggunakan data dummy/simulasi

---

## 6. Output Akhir Proyek

Proyek dinyatakan selesai apabila:

* Website dapat diakses
* User dapat melakukan login, memilih produk, checkout, dan membuat pesanan
* Fitur utama berjalan tanpa error kritis

---

## 7. Pengelolaan Perubahan Scope

Setiap penambahan atau perubahan fitur harus melalui persetujuan **Project Manager** dan dicatat sebagai perubahan scope proyek.
